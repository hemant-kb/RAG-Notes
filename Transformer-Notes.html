Part 0

Prerequisites

Two ideas you must own before the story begins: the context vector (the thing the whole Transformer story is built to fix) and word embeddings (the numbers everything operates on).

Prerequisite 1

What is a Context Vector?

What

A context vector is a single fixed-length list of numbers that is meant to summarise the entire meaning of an input sequence (a sentence). In the classic translation model, the encoder reads the whole source sentence and squeezes everything it understood into one vector c; the decoder then reads only c and produces the output sentence.

c = encode(x1, x2, …, xn)  ∈ ℝd
One vector of dimension d carries the meaning of an n-word sentence — no matter how long n is.
Significance: this is the bottleneck. Forcing a 50-word sentence and a 3-word sentence through the same size vector is the central limitation that attention (Part 2) and ultimately the Transformer were invented to remove.
Why it matters

Everything that follows is a reaction to this one idea. RNN seq2seq has a single context vector → information is lost. Attention lets the decoder build a fresh context vector at every step. Self-attention lets every word build its own context vector from every other word. Keep the phrase "context vector" in mind — the whole architecture is the evolution of how it is built.

🎒
Real-world analogy

Imagine summarising a whole book onto a single index card, handing only that card to a friend, and asking them to rewrite the book in another language from the card alone. For a paragraph it works; for a 400-page novel the card cannot hold enough. That card is the context vector.

Prerequisite 2

Word Embeddings — In Detail

What

A word embedding is a dense vector of real numbers that represents a word's meaning in a continuous space. Words with similar meaning sit close together; the geometry of the space encodes relationships. This replaces the old one-hot representation, where every word is a giant vector of zeros with a single 1 — vectors that are all equidistant and carry no meaning.

One-hot vectors are orthogonal and meaningless; dense embeddings place related words close together.
How — the embedding space

Each dimension is a learned latent feature (loosely: "royalty", "gender", "plural", …). The model learns these numbers from data so that the distance between vectors reflects semantic similarity. Two famous properties:

  • Similarity: related words have high cosine similarity (small angle).
  • Vector arithmetic: directions encode relationships, so king − man + woman ≈ queen holds geometrically.
Worked example — cosine similarity

Take toy 3-dim embeddings king = [0.9, 0.7, 0.1] and queen = [0.8, 0.9, 0.15]. Cosine similarity measures the angle between them:

cos(a, b) = (a · b) / ( ‖a‖ · ‖b‖ )
1
Dot product a · b
0.9·0.8 + 0.7·0.9 + 0.1·0.15 = 0.72 + 0.63 + 0.015 = 1.365
2
Magnitudes
‖king‖ = √(0.81+0.49+0.01) = √1.31 = 1.1446
‖queen‖ = √(0.64+0.81+0.0225) = √1.4725 = 1.2135
3
Cosine
1.365 / (1.1446 × 1.2135) = 1.365 / 1.3890 = 0.9828
≈ 0.98 → the two words point almost the same way → strongly related. ✓
💡
Why embeddings are the foundation

A Transformer never sees letters or words — it only ever manipulates embedding vectors. Every mechanism in this handbook (attention scores, multi-head, FFN) is linear algebra on top of these vectors. Get embeddings, and you can follow everything that follows.

Part 1

The Encoder–Decoder (Seq2Seq)

The model that came before Transformers. Understand its mechanism — and the single flaw that motivated everything else.

Chapter 1

Seq2Seq: Mechanism & the Bottleneck

What

A sequence-to-sequence model maps an input sequence (e.g. an English sentence) to an output sequence (e.g. its French translation). It has two RNNs (LSTM/GRU):

  • Encoder: reads the input word-by-word, updating a hidden state. Its final hidden state becomes the context vector c.
  • Decoder: starts from c and generates the output word-by-word, feeding each predicted word back in as the next input.
How — the math (basics)
Encoder:  ht = tanh( Wx xt + Wh ht−1 )    c = hn
Decoder:  st = tanh( Wy yt−1 + Ws st−1 )    s0 = c
Output:  yt = softmax( Wo st )
The decoder is seeded once with c, then runs on its own. Every output word depends on c only indirectly through s.
Seq2Seq math flow — the encoder recurrence collapses into one context vector c that seeds the decoder recurrence.
Everything the decoder ever knows about the source must fit inside the single context vector c.
Worked example (toy 2-dim hidden states)

Suppose after reading "the cat sat" the encoder's hidden states evolve and the final state is c = h₃ = [0.6, −0.2]. The decoder is initialised with s₀ = c and must produce "le chat dort" using only that one vector as its memory of the source. If the sentence were 40 words long, c would still be just 2 numbers (or 512 in practice) — the later words get crushed.

Why it fails — the problems

✗ Problems with one context vector

  • Fixed-length bottleneck: long sentences overflow a fixed vector → information loss.
  • Recency bias: the final state remembers the end of the sentence best; early words fade.
  • No alignment: when generating word 7, the decoder can't focus on the specific source word that matters — it only has the blurred summary.
  • Sequential: RNNs process one word at a time → slow, hard to parallelise.

✓ The fix: Attention

  • Keep all encoder states h₁…hₙ, not just the last.
  • At each decode step, build a fresh context vector by weighting the encoder states by relevance.
  • The decoder can now "look back" and align each output word to the right input words.
Part 1 summary — Seq2Seq squeezes a whole sentence into one context vector, then unrolls it into the output. The fixed-size vector is a hard bottleneck for long sequences and gives no alignment. The fix — let the decoder read all encoder states and weight them — is attention, next.

Part 2

Attention (Bahdanau & Luong)

The breakthrough: instead of one frozen context vector, build a new one at every step by paying attention to the most relevant input words.

Chapter 2

Attention Mechanism

What

Attention lets the decoder, at each output step t, look at every encoder hidden state and compute a weighted average of them — where the weights say "how relevant is each input word to what I'm generating right now". That weighted average is a per-step context vector ct.

Why

It dissolves the bottleneck. There is no longer one fixed summary; the decoder reconstructs a custom summary for each word it writes. When translating "dort" it can focus on "sat"; when translating "chat" it can focus on "cat" — alignment emerges automatically.

How — Bahdanau vs Luong

Both compute a relevance score between the decoder state and each encoder state, softmax the scores into weights, and take the weighted sum. They differ in the score function:

VariantScore functionNicknameNotes
Bahdanau (2014)v⊤ tanh(W1st + W2h̄s)AdditiveA small neural net scores the pair; more parameters.
Luong · dotst⊤ h̄sMultiplicativeCheap dot product; needs same dimension.
Luong · generalst⊤ W h̄sMultiplicativeLearns a weight matrix W between the two spaces.
scoret,s = st⊤ h̄s   (Luong dot)
αt,s = softmaxs( scoret,s ) = escoret,s / Σs' escoret,s'
ct = Σs αt,s · h̄s   (weighted context)
s̃t = tanh( Wc [ ct ; st ] ) → yt = softmax(Wo s̃t)
Significance of each piece: the dot product is a similarity detector (big when the decoder state and an encoder state point the same way). Softmax turns raw similarities into a probability distribution that sums to 1. The weighted sum pulls together the relevant input words into one context vector for this step.
Attention math flow — score every encoder state against the decoder state, softmax to weights, take the weighted sum to build a fresh context vector each step.
At step t the decoder weights all encoder states; the heaviest weight (here h̄₁) dominates the context vector.
Worked example — Luong dot attention

Decoder state st = [1, 0, 1]. Three encoder states: h̄₁=[1,0,1], h̄₂=[0,1,1], h̄₃=[1,1,0].

1
Scores = dot product of sₜ with each h̄ₛ
s·h̄₁ = 1+0+1 = 2  |  s·h̄₂ = 0+0+1 = 1  |  s·h̄₃ = 1+0+0 = 1
scores = [2, 1, 1]
2
Exponentiate
e² = 7.389,  e¹ = 2.718,  e¹ = 2.718  →  sum = 12.826
3
Softmax → weights αₜ
[7.389, 2.718, 2.718] / 12.826 = [0.576, 0.212, 0.212]
h̄₁ wins — it's most aligned with the decoder's current need.
4
Context vector = weighted sum of h̄ₛ
cₜ = 0.576·[1,0,1] + 0.212·[0,1,1] + 0.212·[1,1,0]
= [0.576+0+0.212, 0+0.212+0.212, 0.576+0.212+0] = [0.788, 0.424, 0.788]

✓ Pros

  • Removes the fixed-vector bottleneck — handles long sentences.
  • Learns alignment (interpretable heatmaps).
  • Big BLEU gains over vanilla seq2seq.

✗ Cons (still RNN-based)

  • Encoder/decoder are still sequential RNNs → no parallelism, slow on long inputs.
  • Attention only connects decoder→encoder, not within a sentence.
Bahdanau et al. 2014 — Neural MT by Jointly Learning to Align and Translate Luong et al. 2015 — Effective Approaches to Attention-based NMT
Part 2 summary — Attention builds a fresh context vector per step as a softmax-weighted sum of all encoder states. Score → softmax → weighted sum. It fixes the bottleneck and gives alignment — but it's still bolted onto sequential RNNs. The next leap: drop the RNN entirely and let a sentence attend to itself.

Part 3

Self-Attention: Static → Dynamic

The heart of the Transformer. Let every word build its own context from every other word — turning static embeddings into dynamic, context-aware ones.

Chapter 3

Self-Attention & Scaled Dot-Product

The progression
The problem with static embeddings

A plain embedding gives a word one vector regardless of context. But meaning depends on context:

🏦
One word, two meanings — same vector

"I sat on the river bank." vs "I deposited cash at the bank." A static embedding gives bank the identical vector in both — it cannot tell finance from geography. We need a vector that changes with its neighbours. That is what self-attention produces.

What is self-attention

Every token looks at every other token in the same sentence and rebuilds itself as a weighted blend of all tokens. "Self" because the queries and the keys/values all come from the one sequence. Three learned roles per token, all derived from its embedding x:

  • Query (Q) — "what am I looking for?"
  • Key (K) — "what do I offer / what am I about?"
  • Value (V) — "the actual content I'll pass on if attended to."
Q = X WQ   K = X WK   V = X WV
Attention(Q,K,V) = softmax( Q K⊤ / √dk ) V
X = matrix of token embeddings (one row per token). WQ, WK, WV are learned projection matrices. dk = key dimension.
Why each step: QK⊤ = every query dotted with every key → a similarity score for each token pair. ÷√dk keeps the scores from growing huge as dimension grows (large dot products push softmax into tiny gradients). softmax → each row becomes weights summing to 1. × V → each token's output is the weighted blend of all values.
Self-attention math flow: Attention(Q,K,V) = softmax(QKᵀ/√dₖ)·V — project, score, scale, softmax, blend values.
📐
Why divide by √dk?

If Q and K entries have variance 1, a dot product over dk dimensions has variance dk — so scores scale like √dk. Large scores make softmax nearly one-hot, where its gradient ≈ 0 and learning stalls. Dividing by √dk rescales variance back to ~1, keeping gradients healthy. For the paper's dk=64, that's ÷8.

Worked example — full self-attention on 3 tokens

Three tokens with 4-dim embeddings (rows of X), and tiny projections to dk=dv=2. We compute the new vector for every token.

1
Inputs
X = [[1,0,1,0],  [0,1,0,1],  [1,1,0,0]]  (t₁, t₂, t₃)
After projecting by WQ,WK,WV:
Q = [[2,0],[0,2],[1,1]]  K = [[0,2],[2,0],[1,1]]  V = [[2,1],[1,1],[1,1]]
2
Scores = Q K⊤ (every token vs every token)
Q K⊤ = [[0, 4, 2],  [4, 0, 2],  [2, 2, 2]]
Row 1 = how much t₁ attends to t₁,t₂,t₃. t₁·k₂ = 4 is the biggest → t₁ is most interested in t₂.
3
Scale by √dk = √2 ≈ 1.414
[[0, 2.828, 1.414],  [2.828, 0, 1.414],  [1.414, 1.414, 1.414]]
4
Softmax each row → attention weights A
row1 = [0.045, 0.768, 0.187]
row2 = [0.768, 0.045, 0.187]
row3 = [0.333, 0.333, 0.333]
t₁ mostly attends to t₂; t₂ mostly to t₁; t₃ spreads evenly.
5
Output = A · V (new contextual vectors)
t₁' = [1.045, 1.0]   t₂' = [1.768, 1.0]   t₃' = [1.333, 1.0]
Each token's output now blends information from the tokens it attended to — a dynamic, contextual embedding.
The softmaxed score matrix: each row tells one token how to blend the others into its new representation.

✓ Pros

  • Fully parallel — all tokens processed at once (no recurrence).
  • Constant path length between any two tokens → captures long-range dependencies.
  • Produces context-aware embeddings ("bank" finally disambiguates).

✗ Cons

  • O(n²) cost — every token attends to every token (quadratic in sequence length).
  • No inherent notion of word order (fixed by positional encoding, Part 5).
  • A single attention captures only one kind of relationship → next problem.
🔭
The single-perspective problem

Take "The man saw the astronomer with a telescope." There are two valid readings: (a) the man used a telescope to see, so "with a telescope" attaches to "saw"; (b) the astronomer has a telescope, so it attaches to "astronomer". A single self-attention produces one weight distribution — it can commit to only one of these relationships at a time. We want the model to capture both perspectives simultaneously. The fix: run several attentions in parallel → multi-head attention.

Vaswani et al. 2017 — Attention Is All You Need (§3.2.1 Scaled Dot-Product Attention)
Part 3 summary — Self-attention turns static embeddings into dynamic contextual ones via softmax(QKᵀ/√dk)V. Q/K/V = ask/offer/content; the scaled dot product scores relevance, softmax normalises, the weighted sum of values rebuilds each token. One head sees one relationship — to capture multiple perspectives at once, we go multi-head.

Part 4

Multi-Head Attention

Run several self-attentions in parallel, each in its own subspace, so the model captures several relationships — several "perspectives" — at the same time.

Chapter 4

Multi-Head Attention

What

Multi-head attention runs h independent attention "heads" in parallel. Each head has its own WQ, WK, WV, so each projects the tokens into a different subspace and learns to focus on a different kind of relationship. Their outputs are concatenated and projected back down.

Why

One head = one weight distribution = one perspective (the telescope problem). With h heads, head 1 can attach "telescope" to "saw" while head 2 attaches it to "astronomer"; one head can track syntax, another coreference, another local phrases. The model no longer has to choose a single relationship.

headi = Attention( X WQ(i), X WK(i), X WV(i) )
MultiHead(X) = Concat( head1, …, headh ) WO
Paper config: h = 8 heads, dmodel = 512, so each head uses dk = dv = 512 / 8 = 64. WO mixes the heads back to dmodel.
Significance: total compute ≈ same as one full-size attention (because each head is 1/h as wide), but you get h different views for free, then WO learns how to combine them.
Multi-head math flow: run h attentions in parallel subspaces, concatenate, project with WO.
Split into h heads (each a smaller attention) → concatenate → project with WO back to dmodel.
Worked example — two heads, the telescope sentence

Run two heads over "…saw the astronomer with a telescope". Each head produces its own attention row for the word telescope (where does it attach?):

1
Head 1 — "instrument" subspace
telescope attends to → saw: 0.71, astronomer: 0.12, with: 0.17
Reading (a): the telescope was the tool used to see.
2
Head 2 — "possession" subspace
telescope attends to → astronomer: 0.68, saw: 0.14, with: 0.18
Reading (b): the telescope belongs to the astronomer.
3
Concatenate + WO
final(telescope) = Wₒ · [ head1 ; head2 ]
The token's representation now encodes both attachments at once — the single-perspective problem is solved.
Two heads, two simultaneous attachments — the mixed perspective a single head could not represent.

✓ Pros

  • Captures multiple relationship types / perspectives simultaneously.
  • Same total cost as one wide attention (heads are thinner).
  • More stable, richer representations.

✗ Cons

  • Choosing h is a hyperparameter; too many tiny heads can underperform.
  • Some heads end up redundant and can be pruned.
  • Still O(n²) per head in sequence length.
Part 4 summary — Multi-head = h parallel attentions in different subspaces, concatenated and projected by WO. It lets one layer hold several relationships at once (the two telescope readings). Paper: h=8, dk=64. Next gap: attention is order-blind — we must inject position.

Part 5

Positional Encoding

Attention processes all tokens at once, losing word order. We add a smooth, periodic signal so the model can sense position.

Chapter 5

Sinusoidal Positional Encoding

Why it's needed

Self-attention is a set operation: shuffle the tokens and the math gives the same answer (the weighted sum doesn't care about order). But "dog bites man" ≠ "man bites dog". Because the Transformer dropped recurrence to gain parallelism, it lost the natural sense of sequence — we must add order information back explicitly.

What it should be

We add a position vector PE(pos) to each token embedding. A good positional signal should be:

  • Bounded — values stay in a fixed range (so they don't blow up for long sequences).
  • Periodic — repeating waves let the model compare relative offsets.
  • Continuous & deterministic — smooth, and defined for any position (even longer than those seen in training).

Functions that are bounded + periodic + continuous → trigonometric (sine & cosine). That is exactly what the paper uses.

PE(pos, 2i)  = sin( pos / 100002i/d )
PE(pos, 2i+1) = cos( pos / 100002i/d )
pos = position in the sentence (0,1,2,…). i = dimension index pair. d = dmodel. Even dims use sin, odd dims use cos.
Significance: each dimension is a sine wave of a different wavelength (short waves for low dims, very long waves for high dims). Together they give every position a unique fingerprint, and because sin/cos have angle-addition identities, the model can learn to attend by relative position (offset k) with a fixed linear map.
Positional-encoding math flow: build sin/cos waves per dimension, then add the PE vector to the token embedding.
How it's used in the paper

The PE vector is simply added to the token embedding (same dimension dmodel): input = embedding(token) + PE(pos). No extra parameters. Each token now carries both what it is and where it is.

Worked example — d = 4

With d=4 there are two wave-pairs (i=0 and i=1). The divisors are 100000/4=1 and 100002/4=100. Compute PE for positions 0, 1, 2:

0
pos = 0
[ sin(0/1), cos(0/1), sin(0/100), cos(0/100) ] = [0, 1, 0, 1]
1
pos = 1
[ sin(1), cos(1), sin(0.01), cos(0.01) ] = [0.8415, 0.5403, 0.0100, 1.0000]
2
pos = 2
[ sin(2), cos(2), sin(0.02), cos(0.02) ] = [0.9093, −0.4161, 0.0200, 0.9998]
Dim 0–1 (short wavelength) change fast between positions; dim 2–3 (long wavelength) barely move → coarse + fine position info together.
Different dimensions oscillate at different rates; their combination uniquely fingerprints every position.
🧭
Sinusoidal vs learned positional embeddings

The paper tested both and found near-identical quality, but chose sinusoidal because it needs no parameters and can extrapolate to sequences longer than any seen in training. Many modern models instead use learned or rotary (RoPE) positions — same goal, different wave.

Vaswani et al. 2017 — §3.5 Positional Encoding
Part 5 summary — Parallel attention is order-blind, so we add a sinusoidal position vector to each embedding. Sin/cos are bounded, periodic, continuous, and let the model reason about relative positions. Different dims = different wavelengths = unique position fingerprint.

Part 6

Normalization

Keeping activations well-scaled so deep stacks train fast and stably. Why Transformers use Layer Norm, not Batch Norm.

Chapter 6

Batch Norm vs Layer Norm

Why normalize — the benefits
  • Stable training: keeps activations at mean≈0, variance≈1 → gradients don't explode/vanish.
  • Faster convergence: allows higher learning rates.
  • Smoother loss landscape: reduces sensitivity to initialization and "internal covariate shift".
  • Mild regularization: a small noise/averaging effect.

The standard recipe is: subtract the mean, divide by the standard deviation, then rescale with two learned parameters γ (scale) and β (shift) so the layer can undo normalization if needed.

x̂ = (x − μ) / √(σ² + ε)   then   y = γ · x̂ + β
The only difference between BatchNorm and LayerNorm is which axis μ and σ² are computed over.
Normalization math flow. LayerNorm takes μ,σ² over a token's features; BatchNorm over the batch per feature — same formula, different axis.
Same matrix, different normalization axis. BatchNorm couples samples in a batch; LayerNorm treats each token independently.

Worked example — same 2×3 matrix, both ways

Activations (2 samples/tokens × 3 features): [[1, 2, 3], [4, 5, 6]] (using ε≈0 for clarity).

B
Batch Norm (per column)
col means = [2.5, 3.5, 4.5]
col var = 2.25 → std = 1.5
col1: (1−2.5)/1.5=−1, (4−2.5)/1.5=+1
→ [[−1, −1, −1],
   [+1, +1, +1]]
Each feature now has mean 0 across the batch.
L
Layer Norm (per row)
row1 mean = 2, var = 0.667 → std = 0.8165
(1−2)/0.8165 = −1.2247
(2−2)/0.8165 = 0
(3−2)/0.8165 = +1.2247
→ [[−1.225, 0, +1.225],
   [−1.225, 0, +1.225]]
Each token normalized within itself.
💡
Why Transformers pick Layer Norm

BatchNorm's statistics depend on the other samples in the batch — unstable for small batches and awkward for variable-length sequences (padding pollutes the mean). LayerNorm normalizes each token over its own features, so it is batch-size and sequence-length independent — ideal for NLP. In the Transformer it appears as "Add & Norm": LayerNorm(x + Sublayer(x)), combining a residual connection with normalization after every sublayer.

Part 7

The Complete Transformer

Assemble every block into the full encoder–decoder of "Attention Is All You Need", with the paper's exact configuration.

Chapter 7

Putting It All Together

The architecture

The Transformer is a stack of N encoder layers feeding a stack of N decoder layers. Each encoder layer = multi-head self-attention + feed-forward, each wrapped in Add&Norm. Each decoder layer adds a masked self-attention and a cross-attention to the encoder output.

The full Transformer: a self-attending encoder stack feeds keys & values to a masked, cross-attending decoder stack, ending in Linear→Softmax over the vocabulary.

Residual connections — how every sub-layer is wrapped

Each sub-layer (attention and FFN, in both encoder and decoder) is wrapped in a residual (skip) connection then LayerNorm: the sub-layer's input is added back to its output before normalizing. This skip path lets gradients flow straight through, so 6 layers can be stacked without vanishing gradients.

output = LayerNorm( x + Sublayer(x) )
The "+ x" is the residual. Without it, deep Transformers do not train.
One layer: the input x flows into each sub-layer and around it (purple skip); the two meet at ⊕ (Add), then LayerNorm. Two such sub-layers per encoder layer; three per decoder layer.

The layer stack — encoder output flows to every decoder layer

The encoder is 6 layers deep; each feeds the next, and the final encoder output (a 3×512 matrix) is what every decoder layer reads as K, V in cross-attention. The decoder is also 6 layers; only its top output goes to Linear→Softmax.

The 6-layer encoder stack produces one output that is broadcast as K,V into the cross-attention of all 6 decoder layers — the bridge from input to output.

End-to-end worked example — predicting one word

Let's translate "We are friends" → "Hum dost hai" and watch the model predict the word after "⟨SOS⟩ Hum". We trace one decode step from input to output. (Shapes use the paper's real dims; the small numbers reuse the verified attention example so you can follow the arithmetic.)

The seven stages of a single decode step, ending in the predicted token "dost".
1
Encode the source "We are friends"
Tokenize → embed → +PE → 6 encoder layers → encoder output = 3 × 512 (one 512-vector per source token). Computed once and reused as K, V for every decoder cross-attention.
2
Decoder input so far: "⟨SOS⟩ Hum"
Shift-right is already done. Embed + positional → X = 2 × 512 (rows for ⟨SOS⟩ and Hum).
3
Masked self-attention (no peeking ahead)
Q,K,V = X·W (2×64). Scores 2×2, add causal mask so "Hum" sees only {⟨SOS⟩, Hum}, ÷√64, softmax, ×V → concat → ·WO → 2 × 512, then + X (residual) and LayerNorm.

Chapter 8

Feed-Forward Network (FFN)

What

After attention mixes information between tokens, the position-wise feed-forward network processes each token independently through the same two-layer MLP — expanding to a wide hidden size, applying ReLU, then projecting back.

Why

Attention is mostly linear (a weighted average). The FFN injects non-linearity and per-token "thinking capacity", letting the model transform the blended features. It is where a large share of the parameters (and learned knowledge) lives.

FFN(x) = max(0, x W1 + b1) W2 + b2
W1: dmodel→dff (512→2048), ReLU, W2: dff→dmodel (2048→512). Same weights for every position.
Significance: "position-wise" = applied to each token separately (a 1×1 transformation), so it adds no cross-token mixing — that job belongs to attention. Expand → non-linear → compress is a classic feature-refinement pattern.
Worked example (tiny: d=2 → 4 → 2)
1
Token vector in
x = [1, −2]
2
Expand: xW₁+b₁ (to width 4)
say result = [3, −1, 2, −4]
3
ReLU: max(0, ·)
[3, 0, 2, 0]  ← negatives zeroed (the non-linearity)
4
Compress: ·W₂+b₂ (back to width 2)
→ [1.4, 0.2]  the refined token vector
Expand → ReLU → compress, applied identically to every token position.
Chapter 9

Masked Self-Attention

What

Masked (causal) self-attention is ordinary self-attention with one rule: a token at position t may attend to positions ≤ t only — never to future tokens.

Why

The decoder generates the output one token at a time, left to right. During training we feed the whole target sentence at once for speed, but each position must predict the next word using only words before it. If it could peek ahead, it would "cheat" — copying the answer it's meant to predict. The mask enforces this autoregressive property.

How

Before softmax, set the scores for future positions to −∞. After softmax, e−∞ = 0, so future tokens get exactly zero weight.

MaskedAttn = softmax( (Q K⊤ / √dk) + M ) V
M is a triangular mask: Mij = 0 if j ≤ i (allowed), −∞ if j > i (future, blocked).
Masked self-attention math flow: add −∞ on the upper triangle before softmax so each token attends only to itself and the past.
Worked example — reuse the 3-token scaled scores

Scaled scores from Part 3 were [[0, 2.828, 1.414], [2.828, 0, 1.414], [1.414, 1.414, 1.414]]. Apply the causal mask (block the upper triangle), then softmax each row:

Chapter 10

Cross-Attention (Encoder–Decoder Attention)

What

Cross-attention is where the decoder finally reads the source sentence. The Query comes from the decoder (what the decoder is trying to produce now); the Keys and Values come from the encoder's output (the encoded source). It is the direct descendant of the Bahdanau/Luong attention from Part 2 — now inside the Transformer.

Why

Masked self-attention only lets the decoder look at what it has generated so far. To translate, it must also look at the input. Cross-attention is the bridge: it aligns each output token to the relevant source tokens — "dort" looks back at "sat".

CrossAttn = softmax( Qdec Kenc⊤ / √dk ) Venc
Q from decoder states, K & V from encoder output. No mask — the decoder may see the entire source.
Significance: this is the only place information flows from encoder to decoder. The same K,V (encoder output) are reused by every decoder layer.
Cross-attention math flow: the decoder's Q queries the encoder's K,V; the score matrix is target×source (the alignment).
Worked example — one decoder query over 3 encoder tokens

Decoder query q = [2, 0.5]. Encoder keys K=[[0,2],[2,0],[1,1]], values V=[[2,1],[1,1],[1,1]], dk=2.

1
Scores = q · each key
q·k₁=0+1=1  |  q·k₂=4+0=4  |  q·k₃=2+0.5=2.5
2
Scale by √2
[0.707, 2.828, 1.768]
3
Softmax → alignment weights
[0.082, 0.682, 0.236]  ← decoder aligns mostly to source token 2
4
Context = weighted sum of encoder values
0.082·[2,1] + 0.682·[1,1] + 0.236·[1,1] = [1.082, 1.0]
This source-aware vector feeds the decoder's next block — translation grounded in the input.
Cross-attention: the decoder's query weights the encoder's tokens, pulling the relevant source content into the decoder.

The whole pipeline in one breath

Source → encoder (embeddings + PE → self-attention → FFN, ×6) → encoder output. Then for each output token: decoder (masked self-attention over what's written → cross-attention into the encoder → FFN, ×6) → Linear → Softmax → next word → feed it back. Repeat until <end>. Every block is just the scaled dot-product attention and tiny MLPs you now know, stacked with residual Add&Norm.

Interview questions
Why does the decoder need a mask but the encoder doesn't?
The encoder sees the whole input at once (no cheating possible — it's the given source). The decoder generates left-to-right and must not see future target tokens it is supposed to predict, so its self-attention is causally masked.
What's the difference between self-attention and cross-attention?
In self-attention Q, K, V all come from the same sequence. In cross-attention the Query comes from the decoder while Keys and Values come from the encoder output — it's how the decoder reads the source.
Why divide attention scores by √dk?
Dot products grow with dimension (variance ≈ dk). Large scores saturate softmax into a near one-hot with vanishing gradients. Scaling by √dk keeps variance ≈ 1 and gradients healthy.
Why multi-head instead of one big attention?
Different heads attend in different subspaces, capturing several relationships at once (e.g. both readings of "…with a telescope"). Total cost is similar because each head is narrower.
Why positional encoding?
Attention is order-invariant (a set operation). Adding sinusoidal position vectors restores word order while staying bounded, periodic, and able to extrapolate to longer sequences.
Part 8 summary — A decoder layer = masked self-attention (no peeking ahead) → cross-attention (read the source, Q=dec/K,V=enc) → FFN (per-token non-linear refine), each in Add&Norm, ending in Linear→Softmax over the vocabulary. Generate one token, feed it back, repeat. That autoregressive loop over the blocks you've learned is the entire Transformer at work.

The one rule for tracking shapes

A matrix multiply (a × b)·(b × c) = (a × c) — the inner dimensions must match and cancel. Every state here follows it: Q·Kᵀ cancels dk (=64) leaving token×token; A·V cancels the token dim leaving token×64; concat·WO maps 512→512. Master that and you can derive any shape in the Transformer yourself.

Decoder adds masked self-attention (4×4 causal) and cross-attention (scores 4×3, target×source from the encoder), then Linear→Softmax → next token, looped autoregressively until ⟨EOS⟩.
